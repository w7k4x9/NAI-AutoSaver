# NAI 자동저장

NovelAI 이미지 페이지(`https://novelai.net/image`)에서 작동하는 Chrome Manifest V3 확장입니다.
자동 생성 · 대기열 · 조합 생성 · 제목 기반 자동 저장을 지원합니다.

## 핵심 기능

- **자동 생성**: 지정한 횟수만큼, 지정한 주기(초)로 NovelAI `Generate` 버튼을 자동 클릭
- **완료 이미지 자동 저장 (기본 ON)**: 생성될 때마다 이미지를 자동 다운로드
- **제목 기반 저장 파일명**: 저장 이름이 `1_b_c`면 `1_b_c (1).png`, `1_b_c (2).png` … 순서로 저장 (NovelAI 폴더 아래)
- **대기열 (Base / 캐릭터 프롬프트 분리)**: 서로 다른 (저장 이름 · Base Prompt · 캐릭터 프롬프트 · 횟수) 항목을 줄 세워 무인 자동 생성
  - `현재 프롬프트 추가`: 지금 NovelAI의 Base 프롬프트 + 캐릭터 프롬프트 + 저장 이름 + 횟수를 항목으로 저장
  - 항목마다 **저장 이름 · Base Prompt · 캐릭터 프롬프트 · 횟수**를 따로 입력/편집, 순서 변경(↑↓), 삭제
  - **Base는 베이스 칸, 캐릭터는 NovelAI 캐릭터 박스에 직접 입력됩니다.** (`;;` 병합 방식은 캐릭터 박스가 이미 있으면 NovelAI가 무시하므로 폐기) 캐릭터 박스가 모자라면 `+ Add Character`를 자동으로 눌러 추가합니다.
  - 한 항목에 여러 캐릭터는 캐릭터 칸에서 `;;`로 구분 → 박스 1·2·…에 순서대로 입력
  - `대기열 실행` → 항목1의 Base/캐릭터 적용 → 횟수만큼 생성·저장 → 항목2 적용 → … 자동 진행
  - **저장 이름은 항목마다 따로 적용됩니다.** ‘자동 생성’ 섹션의 저장 이름은 단발 실행 전용이며 대기열에는 적용되지 않습니다.
  - 펼친 대기열은 **가로 스크롤 카드(위) + 큰 편집창(아래)** 구조입니다. 카드를 누르면 아래에 저장 이름 · Base Prompt · 캐릭터 프롬프트를 큼직한 입력란으로 편집할 수 있고, 실행 중인 항목은 자동으로 보이게 스크롤됩니다
  - 항목별로 저장 파일명이 그 항목의 저장 이름으로 `(1)`부터 매겨짐
  - `반복` 켜면 끝까지 돈 뒤 처음부터 다시 반복
  - 모든 항목 완료 시 알림·사운드, 대기열 JSON `내보내기`/`가져오기`
- **조합 생성기**: Base 1개, 캐릭터 N개(코드·외형·표정세트), 배경 M개(코드·배경태그)를 입력하면 모든 (캐릭터 × 배경 × 표정) 조합을 대기열 항목으로 자동 생성합니다. 항목 제목은 `캐릭터코드_배경코드_감정번호`(예: `w_h_1`), 캐릭터 프롬프트는 `외형 / 표정 / 배경` 3블록으로 구성됩니다. `현재 큐에 추가` · `새 큐로 교체` · `JSON 내보내기`(파일당 최대 항목 수로 분할 가능)를 지원하며, 표정 줄 앞 번호는 자동 제거됩니다.
- **iOS 리퀴드 글래스 UI**: 반투명 글라스 패널 · 광택 하이라이트 · 글라스 토글 (드래그 이동 · 접기/펴기), 글라스 앱 아이콘

## 사용법

1. NovelAI 이미지 페이지 오른쪽 아래에 패널이 뜹니다. (작은 알약을 누르면 펼쳐짐)
2. 프롬프트는 NovelAI 자체 입력칸에서 작성합니다.
3. 단발 자동 생성: `저장 이름`, `횟수`(0=무제한), `주기(초)`를 정하고 `자동 생성 시작`.
4. 대기열: `현재 프롬프트 추가`로 항목을 만든 뒤, 각 항목 카드에서 **저장 이름 · Base Prompt · 캐릭터 프롬프트 · 캐릭터 네거티브 · 횟수**를 따로 채웁니다 → `대기열 실행`. (Base/캐릭터를 비우면 추가 당시 NovelAI 프롬프트가 사용됩니다)
   - **네거티브는 베이스/캐릭터 두 갈래입니다.** 베이스 네거티브는 베이스 Undesired Content로, 캐릭터 네거티브는 각 캐릭터의 Undesired Content로 들어갑니다. 자동생성·대기열 항목·조합생성 각각에 베이스 프롬프트/베이스 네거티브/캐릭터 프롬프트/캐릭터 네거티브 4종을 직접 채울 수 있고, 비우면 현재 NovelAI 값을 그대로 둡니다.
   - **여러 캐릭터**는 캐릭터 프롬프트와 캐릭터 네거티브를 `;;` 로 구분하면 됩니다. NovelAI에 캐릭터 박스가 모자라면 자동으로 `Add Character`를 눌러 채웁니다. `;;` 순서대로 1번 캐릭터 → 첫 박스, 2번 → 두 번째 박스에 매핑됩니다.
5. 멈추려면 `자동 생성 중지` / `대기열 중지`를 누릅니다. 패널이 접혀 있으면 접힌 패널의 빨간 ■ 버튼을 눌러 **패널을 펼친 다음** 중지 버튼을 누르면 됩니다. (빨간 ■ 버튼 자체가 바로 중지시키는 것이 아니라, 펼친 뒤 중지하는 흐름입니다.)
   - 생성 도중에 중지를 누르면 곧바로 끊지 않고 **"정지 대기 중"** 상태로 들어가, 지금 뽑고 있는 마지막 이미지까지 저장한 뒤 정지합니다. 기다리지 않고 즉시 끊으려면 중지를 한 번 더 누르세요.

저장 위치는 브라우저 다운로드 폴더 안의 하위 폴더이며, 설정의 `저장 폴더`에서 바꿀 수 있습니다(기본값 `NovelAI`, 예: `NovelAI/프로젝트A`). 같은 이름이 있으면 Chrome이 자동으로 번호를 덧붙입니다.

## 로드 방법

1. Chrome에서 `chrome://extensions`를 엽니다.
2. 개발자 모드를 켭니다.
3. "압축해제된 확장 프로그램을 로드"를 누릅니다.
4. 이 폴더를 선택합니다.

## 검증

```bash
node --check queue-store.js
node --check content.js
node --check background.js
node --check popup/popup.js
node tests/queue-store.test.js
```

## 변경 이력

### v1.0.10
- **NAI DOM 업데이트 대응**: NovelAI가 캐릭터 프롬프트 박스 클래스를 `prompt-input-box-character-prompt` → `prompt-input-box-character-prompts-1`(복수형 + 번호 접미사)로 바꾸면서 캐릭터 박스를 못 찾던 문제를 수정. 이제 옛/새 클래스명을 모두 인식(`[class*="prompt-input-box-character-prompt"]`).
  - 자동 생성에서 캐릭터 네거티브가 적용되지 않던 문제 해결
  - `현재 프롬프트 추가` 시 캐릭터 네거티브가 베이스 프롬프트 칸으로 밀리고 캐릭터 칸이 비던 변수 밀림 해결
  - 베이스 프롬프트 탐색이 캐릭터 박스를 절대 집지 않도록 방어 로직 추가
- **조합 생성에 `배경 네거티브` 칸 추가**: 기존엔 배경 입력이 `배경 태그`(긍정) 한 칸뿐이라 네거티브를 넣을 곳이 없었음. 이제 배경별 네거티브를 따로 입력하면 해당 항목의 캐릭터 Undesired Content에 합쳐짐. `배경 태그`는 종전대로 긍정 프롬프트로 캐릭터 프롬프트에 합쳐짐.

### v1.0.11
- **캐릭터/베이스 읽기·쓰기를 탭 전환 방식으로 전면 수정**: NovelAI는 캐릭터의 `Prompt` 탭과 `Undesired Content` 탭이 **같은 박스(`prompt-input-box-character-prompts-N`)를 공유**하고 탭에 따라 내용만 바뀐다. 클래스만으로는 프롬프트/네거티브를 구분할 수 없어, 이제 읽고 쓸 때마다 해당 탭을 직접 클릭해 활성화한 뒤 그 박스를 처리한다.
  - 자동 생성에서 캐릭터 네거티브가 적용되지 않던 문제 해결 (캐릭터 UC 탭을 직접 열어 기록)
  - `현재 프롬프트 추가` 시 캐릭터 프롬프트 칸에 네거티브 값이 들어가던 변수 섞임 해결 (프롬프트/네거티브 각각 해당 탭을 열어 읽음)
  - 베이스 네거티브가 빈칸으로 읽히던 문제 해결 (Base Prompt 탭이 켜져 있어도 UC 탭으로 전환해 읽은 뒤 원래 탭 복원)
- **조합 생성: 외형/표정/배경 태그 끝에 콤마 자동 삽입**: 각 그룹을 줄로 쌓을 때 끝에 콤마가 없으면 NAI가 경계 태그를 붙여버리므로(예: `long hair smiling`), 그룹마다 콤마를 하나씩 강제로 붙인다. 이미 콤마로 끝나면 중복으로 붙이지 않는다.
